# GC9307 正常界面（功能说明）

适用硬件：`tps-sw`。  
规范基线：`docs/specs/j9twf-gc9307-normal-ui/SPEC.md`（本文为面向使用者的解释，不替代规格基线）。

---

## 1. 这块屏幕显示什么？

屏幕分为左右两列，对应屏幕下方两个接口：

- 左列：USB‑A（左边接口）
- 右列：USB‑C 高压 / 复用输出通道（右边接口）

右列顶部的 mode / badge 仅用于帮助快速判断当前输出语义：

- 自动跟随协商：显示 `PD` / `PPS` / `DC`
- 手动 TPS 输出：左 badge 显示手动设定电压（如 `3.30V`）
- 手动 `force`：右 badge 显示 `FOCUS`
- 其它手动路径：右 badge 按真实 VBUS 导通状态显示 `ON` / `OFF`

每列从上到下三行分别表示：

1) **电压（V）**  
2) **电流（A）**  
3) **功率（W）**

数值来自对应口的 INA226 电参量监测器，其中功率读取 INA226 的 Power 寄存器（非由电压×电流计算）。

---

## 2. 显示频率

- 屏幕每 **500ms** 刷新一次。

---

## 3. 如何解读特殊显示？

正常数值为定宽格式（总是 6 字符），末尾带单位：

- `5.000V`、`20.00V`、`100.0W` 等

特殊状态：

- `--.--V / --.--A / --.--W`：该端口判定为**未插入**
- `ERROR `：端口已插入，但该项读数读取失败或非法
- `OVER  `：端口已插入，但该项读数超量程（≥1000.0）

颜色提示（便于快速识别）：

- 正常数值：电压为黄色系、电流为红色系、功率为绿色系
- 未插入：灰色
- `ERROR `：红色
- `OVER  `：橙色

---

## 4. “未插入”是怎么判定的？

- USB‑A：若测得电压 `< 1.0V` → 未插入
- USB‑C/PD：自动跟随时，实测电压达到 3V 且实测电流超过 10mA，或 SW2303
  显示 CC/设备在线已捕获、真实协议已协商时，判定为已插入；显示的 V/A/W
  始终来自 U17 INA226 实测值
- 手动 TPS 输出：屏幕仍显示 U17 INA226 实测的 V/A/W；即使 badge 为 `FOCUS`
  或 `OFF`，也不显示目标值或请求值来代替实测

---

## 5. 限制与注意事项

- 本界面不显示 PDO、请求值等深度协议细节；只保留 mode / badge 这种最小语义提示，
  电参量仍以实时实测为准。
- 若出现 `OVER  `，通常意味着异常情况（例如测量/校准错误或硬件故障），建议优先排查。
