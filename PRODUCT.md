# 产品

## 说明

IsolaPurr USB Hub 是一款面向桌面工作台使用的可控 USB Hub 产品，配套 Web / Desktop 控制台用于发现、连接、配置和操作设备。

## 用户

主要用户是在桌面工位上使用 IsolaPurr USB Hub 的硬件与嵌入式工程师。他们把设备接入工作台流程，用来管理 USB 设备、观察供电行为、配置网络、完成固件更新，并可靠地操作每个端口的供电与数据连接。

## 产品目标

IsolaPurr USB Hub 是一个带 Web / Desktop 控制台的 ESP32-S3 USB Hub 产品。控制台需要让产品状态清晰可见、可控：通过 USB 或 Wi-Fi 发现并连接 Hub，应用固件更新，配置 Wi-Fi，查看实时电压、电流、功率数据，并在不猜测设备状态的情况下执行端口级操作。

成功意味着用户可以把 Hub 接到工作台上，快速理解当前连接路径，在可用时通过 Web App 走 Wi-Fi 连接产品，安全地更新或配置产品，并从一个一致的控制台里验证端口行为。界面应减少串口选择、网络凭据、固件版本和供电操作带来的误操作。

## 品牌气质

精准、技术化、冷静。产品应当像一台严肃的工作台仪器配上一套现代桌面控制台：标签直接，状态密集，反馈迅速，不做装饰性表演。

## 反向参考

避免营销式 SaaS dashboard、装饰性卡片网格、巨大的 hero 区、霓虹终端风、玩具化硬件 App，以及会遮蔽状态的动效。不要让硬件状态看起来比实际更模糊、更轻飘。

## 设计原则

- 硬件状态优先：连接、固件、Wi-Fi、遥测和安全状态必须先于次要信息出现。
- 一个工作流表面：设置、通信、更新和控制应当像同一个工作台控制台，而不是几组互不相关的页面。
- 风险显式化：破坏性或会中断设备的操作需要清晰标签、状态反馈，并在必要时确认。
- 密集但冷静：为专家用户显示足够信息，但避免把屏幕变成噪音。
- 能力渐进降级：浏览器不支持的能力应明确降级到 Local USB 或联网 HTTP 路径。

## 无障碍

核心工作流目标为 WCAG AA 对比度，并保证键盘可操作。动效应短促且只用于状态反馈，支持减少动态效果偏好。连接、警告、错误和忙碌状态不能只依赖颜色，必须同时使用文字或图标表达。
